
### 1.1 MAJOR  
Incremented when:
- the structure of the specification changes  
- normative content is added or rewritten  
- breaking conceptual changes occur  
- sections are promoted to or removed from standard status  

### 1.2 MINOR  
Incremented when:
- new sections or subsections are added  
- examples, test cases, or vendor profiles are expanded  
- non‑breaking clarifications are introduced  
- the apparel‑safe subset evolves without breaking compatibility  

### 1.3 PATCH  
Incremented when:
- editorial corrections are made  
- formatting or typographical issues are fixed  
- minor clarifications are added  
- non‑normative content is updated  

---

# 2. Version Scope

The version number applies to the **entire Draphera Bible**, not to individual files.  
Normative sections may include their own “Document Version” header for traceability, but the global version is authoritative.

---

# 3. Release Types

### 3.1 Draft  
Early versions under active development.  
Content may change significantly.

### 3.2 Candidate Standard  
Stable content intended for implementation and testing.  
Only clarifications and minor adjustments are allowed.

### 3.3 Recommended Standard  
Mature, validated content.  
Breaking changes require a new MAJOR version.

---

# 4. Version Lifecycle

Each release follows this lifecycle:

1. **Draft creation**  
2. **Internal review**  
3. **Public review**  
4. **Candidate Standard**  
5. **Recommended Standard**  
6. **Maintenance and patching**
